Detailed Description

A single axis value computed from a "negative" and a "positive" button.

This composite allows to arrange any arbitrary two buttons from a device in an axis configuration such that one button pushes in one direction and the other pushes in the opposite direction.

The limits of the axis are determined by minValue and maxValue. By default, they are set to [-1..1]. The values can be set as parameters.

var action = new InputAction();
action.AddCompositeBinding("Axis(minValue=0,maxValue=2")
.With("Negative", "<Keyboard>/a")
.With("Positive", "<Keyboard>/d");

If both buttons are pressed at the same time, the behavior depends on whichSideWins. By default, neither side will win (WhichSideWins.Neither) and the result will be 0 (or, more precisely, the midpoint between minValue and maxValue). This can be customized to make the positive side win (WhichSideWins.Positive) or the negative one (WhichSideWins.Negative).

This is useful, for example, in a driving game where break should cancel out accelerate. By binding negative to the break control(s) and positive to the acceleration control(s), and setting whichSideWins to WhichSideWins.Negative, if the break button is pressed, it will always cause the acceleration button to be ignored.

The values returned are the actual actuation values of the buttons, unaltered for positive and inverted for negative. This means that if the buttons are actual axes (e.g. the triggers on gamepads), then the values correspond to how much the axis is actuated.

enum UnityEngine.InputSystem.Composites.AxisComposite.WhichSideWins

What happens to the value of an AxisComposite if both positive and negative are actuated at the same time.

Enumerator
Neither 

If both positive and negative are actuated, the sides cancel each other out and the result is 0.

Positive 

If both positive and negative are actuated, the value of positive wins and negative is ignored.

Negative 

If both positive and negative are actuated, the value of negative wins and positive is ignored.
